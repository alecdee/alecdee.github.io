Libraries

This page lists the libraries I've worked on to build unileq up from one instruction into an easy to use architecture.


Libraries:

String Printing

Memory

String Printing

Download print.unl or try the live demo here.


print.unl is fundamental to working on other libraries. It defines the print function which allows us to print ASCII strings as well as numerical values. This allows to print diagnostic messages and see what other libraries are actually doing. It can be called like this:


0 ? print 120 32 61 32 0-1 x 10 0

The numbers after print are the ASCII values for "x = ", followed by 0-1 x which prints out the value of [x]. So if [x]=20, then this will get output to the console:


x = 20

In order to convert a plaintext string into ASCII numerical values, we can use toascii.unl. It takes a string from the console and outputs its ASCII values.


> ./unileq toascii.unl enter text: Hello ascii : 72 101 108 108 111

print_demo.unl shows how to use the library. It's also used in the live version linked above.

Unsigned Math

Download uint.unl or try the live demo here.


uint.unl defines all of the usual functions for handling unsigned integer arithmetic and bitwise operations. For example, to compute the bitwise AND of [B] and [C] and store it [A], we would call


0 ? uint.and A B C

All of the functions are fairly well optimized. The table below gives the worst case number of instructions required per function call.

Random Number Generation

Download random.unl or try the live demo here.


random.unl defines a PRNG with a period of 2^64. It provides five functions for handling random numbers:


random.seed num random.jump num random.get dst random.mod dst mod random.shuffle arr len

For instance, to get a random number mod 52 and store it in rand, call:


0 ? random.get rand mod rand: 0 mod: 52

random_demo.unl shows how to use the library. It's also used in the live version linked above.

Memory

Download memory.unl or try the live demo here.


memory.unl defines functions for managing and allocating memory. In order to allocate memory safely, it needs to be last when concatenating with other libraries.


mem.getargs dst0 refs0 dst1 refs1 ... 0 mem.get val mem mem.getidx val mem idx mem.set mem val mem.setidx mem idx val mem.zero dst len mem.copy dst src len mem.alloc mem len mem.realloc ret mem len mem.free mem mem.alloc.verifyfree

For instance, to allocate a block of memory with 30 cells and set mem[2]=400:


0 ? mem.alloc mem len 0 ? mem.set mem idx val 0 ? mem.free mem mem: 0 len: 30 idx: 2 val: 400

memory_demo.unl shows how to use the library. It's also used in the live version linked above.
