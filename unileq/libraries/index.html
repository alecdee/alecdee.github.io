Libraries

This page lists the libraries I've worked on to build unileq up from one instruction into an easy to use architecture.


Libraries:

String Printing
Unsigned Integers
Random Number Generation

String Printing

print.unl is fundamental to working on other libraries. It defines the function print which allows us to print ASCII strings as well as numerical values. This allows to print diagnostic messages and see what other libraries are actually doing. It can be called like this:


0 ? print 120 32 61 32 0-1 x 10 0

The values after print are the ASCII values for "x = ", followed by 0-1 x which prints out the value of [x]. So if [x]=20, then this will get output to the console:


x = 20

In order to convert a plaintext string into ASCII numerical values, we can use toascii.unl. It takes a string from the console and outputs its ASCII values.


> ./unileq toascii.unl enter text: Hello ascii : 72 101 108 108 111

Unsigned Integers

uint.unl defines all of the usual functions for handling unsigned integer arithmetic and bitwise operations. For example, to compute the bitwise AND of [B] and [C] and store it [A], we would call


0 ? uint.and A B C

All of the functions are fairly well optimized. The table below gives the average number of unileq instructions required per function call.


FunctionInstructions
cmp28
set24
neg25
add30
sub31
mul933

or676
xor676

Testing is carried out by uinttest.unl. It needs to be concatenated together with uint.unl and print.unl. The command to do so is in uinttest.unl.

Random Number Generation

random.unl defines a PRNG with a period of 2^64. It provides five functions for handling random numbers:


random.seed num random.jump num random.get dst random.mod dst mod random.shuffle arr len

For instance, to get a random number mod 52 and store it in rand, call:


0 ? random.get rand mod rand:0 mod:52

Testing is carried out by randomtest.unl. It needs to be concatenated together with random.unl and print.unl. The command to do so is in randomtest.unl.
